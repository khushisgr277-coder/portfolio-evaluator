# Contributing to Portfolio Evaluator

First off, thanks for taking the time to contribute!

The following is a set of guidelines for contributing to Portfolio Evaluator. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## Code of Conduct
Please be respectful and constructive in issues and pull requests.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue and include:
- A clear and descriptive title
- Steps to reproduce the issue
- Expected vs actual behavior
- Any relevant logs or errors

### Suggesting Enhancements
Have an idea for a new feature? We'd love to hear it! Open an issue indicating it's a feature request and describe how it works.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Update the documentation if needed.
4. Issue that pull request!

## Code Style
- We use ESLint and Prettier. Run `npm run lint` before committing (if available).
- Keep components small and focused.
- Add brief comments for tricky logic.

Thanks again!
